开放你的博客

Posté le 2017-11-01 | In 业余

1. 注册github账号（已有github的老司机自动略过）

2. 在github上新建一个项目，命名格式为 ‘github名.github.io’


3. 修改根目录下的’_config.yml’文件

4. 在根目录下右键运行Git Bash here

5. 恭喜你，可以开始装逼了！

在浏览器输入’github名.github.io’，come on!

温馨提示 : 博主屌丝尝试去码云部署，历经千难万阻，依旧失败。建议还是乖乖使用github。

2. 小试牛刀

  1. 新建个文档玩玩。找到根目录/source/_posts，在这个文件夹里新建一个md文件’天才第一步.md’,然后开始编辑你的md文件。
  2. 刷新浏览器 :

  3. 会发现’ARTICLES RÉCENTS’下多了个’(no title)’，这就是那个’天才第一步.md’文件，但是显示no title 是不是很别扭，没关系，现在就来配置我们新建的文件。

    在文件最上方添加以’—‘分隔的区域，以下参数我不解释说明的就可不必去管，使用默认值就好

    参数 | 描述
    layout | 布局
    title | 标题
    date | 建立日期
    update | 更新日期
    comments | 开启文章的评论功能
    tags | 标签
    categories | 分类
    permalink | 覆盖文章网址

  4. 配置完成，刷新浏览器

  5. 会发现即使我设置了’comments’属性为true，让别人也参与到对我文章的评论中来（多夸夸我），可是下面还是没有评论功能，这怎么行？
  6. 这个问题可能是由于Hexo默认主题的原因，没关系，换了它。

3. 切换主题

  1. 搜索Hexo主题，一抓一大把，选一个你中意的，去github上下载即可。
  2. 将下载下来的解压之后的文件夹放入根目录/themes目录下
  3. 修改根目录下的’_config.yml’文件
  4. 打开命令行，ctrl+c停掉服务，hexo erver 重启服务，刷新浏览器。
  5. 主题切换成功

4.博客整体的配置

根目录下的_config.yml文件，详情内容见这里

  1. title : 网站标题
  2. subtitle : 网站子标题
  3. description : 网站描述，搜索引擎
  4. author : 作者大名
  5. language : 网站语言
  6. theme : 选用主题，从/themes/下选择
