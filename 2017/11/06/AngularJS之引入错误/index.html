当你的angularJS应用出现以下状况时，不要惊慌，且看博主如何逗比

前言

当我拿到别人给的代码，报错如上，我很牛逼的告诉他，so easy，结果搞了两个小时才解决了本该两分钟就能解决的问题。

我的想法

我看到这个错误的时候，就只有一个想法，很明显是’loginCtrl’控制器引入错误了，一定是loginCtrl的引入有问题。

  • 引入路径错误 : 结果就这么一直改啊改啊改啊改，那（引入loginCtrl的js路径）从这挪到那，从那挪到这，可惜那个红红的一行字还是屹立不倒的杵在那。

  • 指令位置错误 : 后来我又想，是不是因为ng-controller指令放置的位置有问题，就开始那这个指令从这挪到那从那挪到这的挪着，又或者突发奇想，再添一层div加指令，很开心，错误还是在那静静的坐着。

  • 单词拼写错误 : 我又开始怀疑是不是单词拼写错误的原因，很遗憾，并不是。就在我开始思考人生，自己还是不适合干这一行的时候，求教的朋友开始操作起来了，他这一简单的操作，却把我从绝望中拯救出来了。

大神的想法

( 大神们早已看穿了一切，不是吗 )

是啊，他把loginCtrl中的依赖注入项都删除掉。我刚开始还在想，这跟那些没关系，这就是这个控制器的引入错误，怎么会和控制器的依赖注入有关系呢。当我盯着那一行红红的english words瞅的时候，我瞬间有了扇自己一巴掌得念头。

总结

好了，问题解决。心急吃不了热豆腐，==遇到报错，一定要一个单词一个单词的过一遍==( 纵然你可能一个单词也不认识 )。

其实现在想想还是自己的基础太差，

  • 控制器引入报错的话，就不会是报这个错，而是报loginCtrl is not define的错。
  • 报这个错，最后的单词是指明错误所在的区域，错误是依赖注入的报错。
